Grid system

The backbone of foundation is the grid system. This system enables a programmer to write a website that is compatible with both mobile and desktop browsers. There are three different parts to this system; the mobile grid, the desktop grid, and the block grid. HTML div elements are what Foundation uses as the building blocks for a website. Div elements hold the different types of website elements that a programmer may want to display and they are selected using the class attribute. A div element with the class “row” will be a container for columns of a grid. Inside this element, we are able to nest other div elements with the class “column” that will serve as columns in a table. By default, Foundation divides the screen into 12 blocks across, and this tutorial will do the same.

Mobile Grid

The mobile grid is the type of grid that will be displayed when the browser window that the website is being displayed on is 768px wide or less. In order to specify the number of blocks wide you want your column element to be on a small screen like this, you use the “small-#” class, where # is a number from 1 to 12. All of the column widths of the columns in a row must add up to 12 in order for the table to be centered within the web page. For example, say we want to make two columns, the first of which is 4 blocks wide, and the second is 8 blocks wide. We will use the following code snippet:

<div class=”row”>
    <div class=”small-4 columns”>Column 1</div>
    <div class=”small-8 columns”>Column 2</div>
</div>

The 4 and 8 specify the width of the column, and the word “small” specifies that this size is for when the screen is less 768px wide or less. The next aspect to cover is what to do when the screen is larger than 768px. This is where the desktop grid comes into the picture.

Desktop Grid

The desktop grid is just like the mobile grid except it is the grid that is displayed when the screen is larger than 768px wide. Instead of “small” in the class name, we will use the word “large.” Here is an example of code that will generate one column that is 5 blocks wide and another that is 7 blocks wide on a large screen:

<div class=”row”>
    <div class=”large-5 columns”>Column 1</div>
    <div class=”large-7 columns”>Column 2</div>
</div>

Again, the number of blocks adds up to 12 for this row, which ensures that the content will effectively fill the entire width of the screen. The mobile grid and the desktop grid are both useful on their own, but they reach their full potential when used together.

Mobile and Desktop Working Together

Using the mobile grid and desktop grid together ensures that a page will be look great on both small and large screens. In order to put these two concepts together, all that needs to be done is use both the “small” and “large” keywords within the class of a particular column element. Below is an example of code that will generate two columns. On a small screen, the first column will be 4 blocks wide, while on a large screen is will be 5 blocks wide. The second column will be 8 blocks wide on a small screen and 7 blocks wide on a large screen.

<div class=”row”>
    <div class=”large-5 columns”>Column 1</div>
    <div class=”large-7 columns”>Column 2</div>
</div>

Again, the number of blocks adds up to 12 for this row, which ensures that the content will effectively fill the entire width of the screen. The mobile grid and the desktop grid are both useful on their own, but they reach their full potential when used together.

Block Grid

The block grid is very similar to the mobile and desktop grid system, except that it resizes content based on a fixed number of elements across. For example, if you wanted to display a row of 3 images that each took up ⅓ of the screen, the block grid would be the solution. As the mobile and desktop grid used div element for the row and columns, the block grid uses an unordered list. Another similarity between these two systems is the “small” and “large” keywords. Here is some code that will display 8 images. On a large screen, each image will take up ¼ of the screen, so there will be 4 across. On a small screen, each image will take up ½ of the screen, so there will be 2 across.

<ul class="small-block-grid-2 large-block-grid-4">
    <li><img src="image1.jpeg"></li>
    <li><img src="image2.jpeg"></li>
    <li><img src="image3.jpeg"></li>
    <li><img src="image4.jpeg"></li>
    <li><img src="image5.jpeg"></li>
    <li><img src="image6.jpeg"></li>
    <li><img src="image7.jpeg"></li>
    <li><img src="image8.jpeg"></li>
</ul>

Here, as the screen size changes, as long as it is less than or equal to 768px wide, 2 images will fill the screen width, resizing the images to achieve this. When the screen is larger than this, 4 images will take up the width and be resized automatically to fill the width.

Buttons

Adding neatly styled and colorful buttons is made very easy with Foundation. Buttons can easily be added with an anchor tag with a button class. This generates a simple button with the text displayed on the button. Adding more classes to this will style the buttons even more. For example, adding a tiny, small, large, expanded, round and radius classes will style the buttons accordingly. Also adding classes like success (green), secondary (gray), alert (red) etc… adds color to the button. By listing the buttons as list items in an unordered list with class button-group will make a button group.

Plugins

Foundation also comes with a lot of plugins that uses JavaScript (a client side programming language). They include tabs, dropdowns, accordions, orbit, abide validation, modals, tooltips etc… These help to make a website to be more functional and organized. Some of these plugins are described in the following sections. For more plugins, the Zurb Foundation official documentation will be very helpful.

Orbit

Traditional ways of adding an image slider is very complicated compared to what Foundation offers. Orbit helps with making image sliders in a very intuitive and simple way. Although orbit is no longer officially supported by (no more updates) Zurb (the group that designed Foundation), it will still be part of the package. To use an orbit, simply add an unordered list with the foundation specific attribute data-orbit. Then, add images as list items. It is also possible to add a caption by adding a div with the class orbit-caption within the list tag for each image. The caption goes within the div tags.
